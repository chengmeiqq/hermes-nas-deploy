## 日常操作

### 启动/停止容器
```bash
cd /vol1/1000/docker/hermes
sg docker -c "docker compose up -d"      # 启动
sg docker -c "docker compose down"        # 停止
sg docker -c "docker compose restart"     # 重启
sg docker -c "docker compose ps"          # 查看状态
```

### 查看日志
```bash
sg docker -c "docker logs hermes -f"           # hermes 网关日志
sg docker -c "docker logs hermes-dashboard -f"  # dashboard 日志
```

### 更新镜像
```bash
cd /vol1/1000/docker/hermes
sg docker -c "docker pull nousresearch/hermes-agent:latest"
sg docker -c "docker compose up -d --force-recreate"
```

### 仪表盘密码修改
1. 生成新密码哈希：
```bash
cd /vol1/1000/docker/hermes
sg docker -c "docker exec hermes-dashboard /opt/hermes/.venv/bin/python3 -c \"
from hermes.plugins.dashboard_auth.basic import DashboardBasicAuthPlugin
print(DashboardBasicAuthPlugin.hash_password('新密码'))
\""
```
2. 编辑 `data/.env`，修改 `HERMES_DASHBOARD_BASIC_AUTH_PASSWORD_HASH`
3. 重启：`sg docker -c "docker compose restart hermes-dashboard"`

## 故障排查

### 容器未启动
```bash
cd /vol1/1000/docker/hermes
sg docker -c "docker compose ps"        # 确认状态
sg docker -c "docker compose up -d"     # 重新启动
```

### Dashboard 无法访问
1. 检查端口是否被占用：`ss -tlnp | grep 9119`
2. 重启容器：`sg docker -c "docker compose restart hermes-dashboard"`
3. 查看日志：`sg docker -c "docker logs hermes-dashboard 2>&1 | tail -50"`

### 飞书连接断开
1. 检查容器状态：`sg docker -c "docker compose ps"`
2. 查看日志：`sg docker -c "docker logs hermes 2>&1 | grep 'lark\|feishu\|connect'"`
3. 重启容器：`sg docker -c "docker compose restart hermes"`

### 模型配置问题
编辑 `data/config.yaml`，确认 Qwen3.6-plus 相关配置：
```yaml
model:
  provider: qwen
  model: qwen3.6-plus
  base_url: <你的API地址>
  api_key: <你的API密钥>
```

## 备份与恢复

### 备份
```bash
cd /vol1/1000/docker/hermes
tar czf hermes-backup-$(date +%Y%m%d-%H%M%S).tar.gz --exclude='hermes-backup-*' .
```

### 恢复
```bash

tar xzf hermes-backup-YYYYMMDD-HHMMSS.tar.gz
cd hermes
docker compose up -d
```

## 自定义镜像

已构建含所有配置的自定义镜像 `hermes-agent:custom` (3.91GB)，可用于快速部署。

如需使用自定义镜像：
1. 将镜像导出：`docker save hermes-agent:custom > hermes-custom.tar`
2. 在新机器导入：`docker load < hermes-custom.tar`
3. 修改 `docker-compose.yml` 中的 `image` 为 `hermes-agent:custom`
